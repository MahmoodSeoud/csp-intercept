# Contributing to csp-intercept

Thanks for looking. csp-intercept is a measuring instrument: its whole value is
that you can trust a number it prints. So the bar for changes that touch the
**loss injector, the oracles, or the determinism of a run** is unusually high —
a subtle bug there doesn't crash, it quietly produces a wrong result that looks
right. The bar for everything else (docs, scripts, plots, tests) is "please open
a PR, the worst that happens is we talk about it."

## What lives where

```
lib/          Pure C, zero-dependency core (fragment indexing, the
              Gilbert-Elliott loss model). Builds and tests anywhere.
tests/        Unit tests (test_lib.c), the GE host harness, e2e/ two-oracle loops.
inject/       The in-path loss injector (ci_inject_bridge): zmq/can ingress ->
              per-fragment drop shim -> egress, with the split-horizon pump.
proxy/        The lossy zmqproxy front-end.
apm/          The CSP monitor APM, dlopen'd into csh (the wire-monitor oracle).
scripts/      Sweep drivers (dipp-sweep, satdeploy-sweep, rawdtp-sweep,
              upload-point) and the matplotlib plotters.
captures/     Run outputs. Reproducible scratch is gitignored; the headline
              evidence (sweeps, rq3_corruption.csv, *_finding.md) is tracked.
figures/      Rendered figures (regenerated by scripts/plot_*.py).
satguard/     The report layer over the capture CSVs.
docs/         HOWTO.md (the commands) + protocol/metric notes.
diagrams/     Test-rig diagrams (excalidraw + mermaid sources, png/svg renders).
subprojects/  Vendored CSH deps as git submodules (csp, slash, param, apm_csh).
```

The pure `lib/` + `tests/` build with **zero dependencies**. Everything that
touches a live bus (the injector, proxy, monitor APM) is behind
`-Dfrontends=true` and needs the vendored submodules plus system `libzmq`.

## Before you start

1. **Pull the submodules.** The front-ends vendor `libcsp`, `slash`, `libparam`,
   and `libapm_csh`. Forgetting this is the most common broken first build.

   ```sh
   git submodule update --init --recursive
   ```

2. **Install native deps** (Debian/Ubuntu):

   ```sh
   sudo apt install libzmq3-dev libsocketcan-dev libbsd-dev pkg-config \
                    python3 meson ninja-build build-essential
   ```

   The pure lib + test suite needs none of these to compile; they're for the
   front-ends and the CAN bench.

## Build and test

```sh
meson setup build -Dfrontends=true   # first time only; omit the flag for lib-only
meson compile -C build
meson test -C build --print-errorlogs
```

Run the test suite after **any** change — it covers the lib units, proxy
determinism, the APM drain, the RDP + DTP two-oracle loops, and the in-path drop
shim. A change that turns the suite red is not done. See `docs/HOWTO.md` for the
live-bus and flatsat recipes.

## House rules

- **Don't weaken an oracle to make a run pass.** The three oracles (drop log,
  wire monitor, sha256) are independent on purpose. If they disagree, that's a
  finding, not a bug to paper over.
- **Determinism is load-bearing.** A run is `(loss, seed)` reproducible. If you
  touch the injector or the GE model, prove the same seed still gives the same
  drop set.
- **Report results honestly.** A pre-registered prediction that gets refuted
  stays reported as refuted. The instrument's credibility is the whole point.
- **Match the surrounding style.** Our own targets are strict C11 + `-Werror`
  (`ci_strict`); the vendored libs keep their own warning tolerance — don't
  globalize our flags onto them.
